Software Testing Assignment

Quadratic Roots and Triangle Type Testing Analysis

This page presents two programs and their complete testing-oriented analysis: boundary value test cases, robust test cases, equivalence class test cases, flow graph, DD path graph, and independent paths from the DD path graph.

Assumptions used for test design: Quadratic coefficients `a`, `b`, and `c` are assumed to be integers in the range `[-100, 100]` with `a != 0` for a valid quadratic equation. Triangle sides are assumed to be integers in the range `[1, 200]`.

Program A: Find the Roots of a Quadratic Equation

The program reads the coefficients `a`, `b`, and `c`, validates the equation, computes the discriminant, and identifies whether the equation has two distinct real roots, equal roots, or complex roots.

Program Logic

START Read a, b, c IF a == 0 Print "Not a quadratic equation" ELSE d = b*b - 4*a*c IF d > 0 Print "Two distinct real roots" ELSE IF d == 0 Print "Two equal real roots" ELSE Print "Complex roots" END

Decision Basis

  • `a == 0` means the equation is invalid as a quadratic equation.
  • `d > 0` gives two distinct real roots.
  • `d == 0` gives equal real roots.
  • `d < 0` gives complex roots.

Program B: Find the Type of a Triangle

The program reads the three sides of a triangle, validates whether a triangle can be formed, and classifies it as equilateral, isosceles, or scalene.

Program Logic

START Read x, y, z IF x <= 0 OR y <= 0 OR z <= 0 Print "Invalid side" ELSE IF x + y <= z OR x + z <= y OR y + z <= x Print "Not a triangle" ELSE IF x == y AND y == z Print "Equilateral" ELSE IF x == y OR y == z OR x == z Print "Isosceles" ELSE Print "Scalene" END

Decision Basis

  • All sides must be positive.
  • Triangle inequality must hold.
  • All equal sides produce an equilateral triangle.
  • Exactly two equal sides produce an isosceles triangle.
  • All unequal sides produce a scalene triangle.
